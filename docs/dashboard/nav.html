▍ Stage 5 — Closed-Loop Navigation

estimator → planner → command4/4 waypointson the estimate

The capstone slice: the robot walks to a sequence of waypoints using only its own onboard state estimate, closing the loop sensors → estimator → waypoint planner → VelocityCommand → gait steering — the same command interface the real control stack accepts.

The loop

Every control step the ESKF ingests the onboard sensors and reports a pose; a WaypointNavigator turns that pose into a body-frame VelocityCommand(vx, vy, vyaw) toward the active waypoint, which steers the walking gait. Because the planner sees only the estimate, estimation error surfaces directly as navigation error — which is exactly why an accurate estimator matters for navigation.

The through-line

The estimated and true paths diverge because heading is unobservable from IMU + contact alone — the same limitation Stage 1 diagnosed and Stage 2's VIO fixes. Navigation makes the consequence concrete and visible: a few degrees of yaw drift becomes tens of centimeters of position error at the goal.

Continuous 3-lap drift with a realistic uncalibrated IMU bias: leg-odometry and the ESKF accumulate unbounded yaw drift (17° / 7°), while VIO stays bounded at ~1° across all three laps as vision keeps pinning the yaw-axis bias. Position error (right) oscillates over the closed loop.

And the VIO-over-ESKF win is robust, not a lucky seed: across 5 seeds VIO beats the ESKF on drift and gyro-z bias on every one (median 0.15% vs 0.49%; bg,z 0.19 vs 1.66 mrad/s).

TakeawayThe whole project reads as one arc — leg-odometry and IMU drift in yaw, VIO anchors it, self-calibration keeps the sensors honest, the embedded-noise study says timing is what breaks on hardware, and navigation closes the loop and shows why every earlier stage mattered.
