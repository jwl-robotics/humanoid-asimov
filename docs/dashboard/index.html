⚠ locomotion honesty — the walking that feeds these sensors is a development stand-in: a PD-tracked open imitation gait plus a torso “virtual support” that permanently offloads ~44% of body weight and holds the torso upright (without it the open-loop gait topples at t ≈ 0.74 s). This is not true dynamic walking — it exists only to produce self-consistent IMU / encoder / contact data + ground truth for estimator development. The estimator never sees the gait targets, the support wrench, or the ground truth. The real RL locomotion policy is the designated next-milestone upgrade. Related: the foot-contact flag is derived from the physics engine's contact force (>1 N threshold) — this robot carries no foot-force sensor, so real hardware needs a contact detector (motor current / joint torque); the estimator consumes a clean boolean, so that swap is interface-compatible.

  6. STAGE 3✅ DONE
    Online Auto-Calib
    camera-mount extrinsic δφHC recovered online (state 21→25; the camera–IMU time-offset td is carried but stays frozen — this constant-rate loop can't excite it) · recovers an injected 3.1° mount error 87% on the observable axis · Jacobians FD-verified · covariance is observability-honest
    ▸ calibration deep dive
  7. STAGE 3B✅ DONE
    Active Calibration
    gentle neck-scan (~5° · 0.3 Hz) while walking, mount + td unfrozen vs injected 3.1° + 10 ms errors · weak-axis funnel σ 7.5→4.2 mrad, error 15.5→7.3 · td recovered to +0.6 ms with NO scan — the gait ripple already excites it · front-end survives the scan (74–77% flat)
    ▸ calibration deep dive
  8. STAGE 4✅ DONE
    Sim-to-Real Noise
    embedded pipeline: quantization · CAN stagger · latency · jitter · finding: TIMING dominates (latency 5 ms → 2.04% drift), amplitude negligible
    ▸ sim-to-real deep dive
  9. STAGE 4B✅ DONE
    Delay Compensation
    source capture timestamps + buffer-and-replay fusion (timing.py): late measurements fuse at their stamp and replay forward · 15 ms latency 6.46→0.23% drift · position NEES 26→2.5 (confidently-wrong → honest) · flat to 20 ms · zero delay reduces bit-for-bit to the naive loop
    ▸ sim-to-real deep dive
  10. STAGE 5✅ DONE
    Nav Slice
    waypoint planner → VelocityCommand(vx, vy, vyaw) on the estimated state · reaches 4/4 waypoints closed-loop · 7.1 cm / 0.59% from truth
    ▸ navigation deep dive
  11. STAGE 5B✅ DONE
    Closed-Loop Showdown
    the same 3-circuit course under four estimator configs, all steering on their own estimate · replay-compensated 5 ms latency restores the clean numbers (terminal yaw 5.7°→3.7°) · the decisive failure is contact over-trust, not latency: σc=0.05 finishes 10/12 @ 98° yaw — in-loop VIO rescues it to 12/12 @ 17°
    ▸ navigation deep dive
